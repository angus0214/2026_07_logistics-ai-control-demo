# 0001. Markdown Rendering for Law Copilot

Date: 2026-07-08

## Status

Accepted

## Context

The Law Copilot outputs responses in Markdown format (e.g., bolding for emphasis, lists for rules, newlines). Currently, the Vue 3 frontend renders this as raw text using Vue's text interpolation, making it unreadable. We need a way to parse Markdown into HTML, inject it into the DOM, and style it beautifully in our Tailwind CSS environment.

## Decision

1. **Parser**: We will use `marked` because it is lightweight, fast, and easy to integrate with Vue 3.
2. **Security**: Since this is an internal POC and we trust the LLM prompts, we will **NOT** use `DOMPurify` to sanitize the output for now. We will directly inject the parsed HTML using `v-html`. This is a conscious trade-off for speed and development agility, to be revisited before production.
3. **Styling**: We will install `@tailwindcss/typography` and apply the `prose prose-invert` classes to restore proper typography (headings, lists, bold text) to the unstyled HTML generated by `marked`. This overcomes Tailwind's aggressive CSS reset (Preflight).

## Consequences

- We can quickly deliver a beautiful, highly readable Copilot UI.
- We accept the risk of XSS if the LLM hallucinates malicious script tags (acceptable in POC).
- The frontend bundle size will increase slightly due to the new dependencies.
